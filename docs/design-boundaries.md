# LogDiet Design Boundaries

LogDiet is original Apache-2.0 code.

It may learn from broad public workflow patterns in the AI coding-agent ecosystem, but it must not copy protected expression from other projects. Do not copy code, docs, examples, fixture logs, benchmark cases, naming, slogans, UI text, installer behavior, or parser logic from other projects.

LogDiet's independent product thesis is:

- AI coding agents waste context on raw terminal output.
- LogDiet preserves raw output locally.
- LogDiet feeds agents compact, expandable evidence instead.

Contributions must use synthetic fixtures. Contributions must describe principles, not copy implementation. Avoid competitor comparison in README hero or marketing copy. Design comparisons, if any, belong in design docs.
